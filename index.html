Критерии качества верстки

Соответствие дизайну

Верстка должна соответствовать дизайну

Кроссбраузерность

Пользователи сайта могут использовать разные браузеры в разных системах, поэтому нужно добиваться единого отображения.

Вёрстка должна идентично отображаться в последних стабильных версиях браузеров Chrome, Firefox, Safari, если иное не указано в техзадании проекта.

Browserstack - может помочь

Адаптивность

Верстка должна выглядеть не сломанной на всех основных разрешениях, начиная с 320px.

Валидность

  • Разметка должна проходить валидацию в валидаторе W3C без ошибок (error).
  • В проверке могут быть предупреждения (warning).

Оптимизированность

  • Выполнены базовые оптимизации Lighthouse(PageSpeed)
  • На всех страницах для контентных изображений используется ленивая загрузка изображений
  • Используются оптимальные форматы изображений и шрифтов
  • Стили и скрипты минимизированы

Accessibility

Как проверить верстку на доступность?

  • Заполнен alt для изображений, если изображение декоративное - оно должно быть скрыто от скринридеров с помощью aria-hidden="true".
  • Все кликабельные элементы должны фокусироваться по табу.
  • Контрастность основных элементов должна быть не меньше 3:1. Соотношение контрастности текстовых элементов можно проверить с помощью браузерного инспектора. Остальные UI элементы можно проверить на специальных ресурсах типа seochecker.it введя значение цвета Foreground и Background.
  • Скринридер должен правильно понимать и читать все основные разделы и элементы на странице.

Используется методология БЭМ

Именование классов

  • Никакого транслита
  • Отсутствие граматических ошибок
  • Логически подходящие названия классов

Переполнение

  • Текст должен оставаться в рамках родительского блока при переполнении
  • Текст не должен смещать другие блоки.
  • Родительский блок должен сохранять минимальные размеры при недополнении.
  • При увеличении количества элементов они должны оставаться в рамках родительского блока.
  • Слова, длиннее ширины родительского блока, должны переноситься или заканчиваться троеточием...

И еще немного

  • В проекте не должно быть закоментированных фрагментов кода без осмысленного текстового пояснения.
  • В проекте не должно быть неиспользуемых файлов.
  • В разметке не должно быть лишних обёрток и элементов, которые используются для оформления и могут быть заменены на псевдоэлементы.
  • Файлы стилей должны быть разделены согласно структуры проекта.
